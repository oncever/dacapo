To use the suite, all you need is the dacapo jar, which can be downloaded here (69MB).

We have a source distribution available here (6 MB).  To build the dacapo jar from these sources you will need the apache 'ant' tool - see the supplied README file for details.

This contains the DaCapo benchmark software but none of the third-party software. The ant task 'sources' downloads the required packages.

NEW Since January 2007, we have made available a binary distribution for use by those doing ahead of time static analysis with tools such as Soot. Note that this is not recommended for other users, as the standard jar file remains the simplest, and standard way to run the DaCapo benchmarks. Please only use the new 'deps' release if you know what you're doing. The new 'deps' distribution consists of a zip archive containing two jar files for each benchmark, one, <bm>.jar containing the core classes for the given benchmark, and the other, <bm>-deps.jar containing the external dependencies for the benchmark bm.
